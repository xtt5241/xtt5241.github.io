常用函数

ntohs()

ntohs函数是用于将16位整数从网络字节序（big-endian字节序）转换为主机字节序（机器的本地字节序）的函数。下面是关于ntohs函数的解释：

  1. 函数原型：

    • netshort：表示网络字节序中的16位整数值，即需要进行转换的数值。

  2. 函数功能：

    • ntohs函数的作用是将16位整数从网络字节序（大端字节序）转换为主机字节序（机器的本地字节序）。
    • 在网络通信中，不同的计算机体系结构使用的字节序可能不同。大部分网络协议规定使用网络字节序（即big-endian字节序）进行数据传输，因此在接收数据时，需要将网络字节序转换为本地字节序以便进行处理。

  3. 参数说明：

    • netshort：表示一个16位整数值，它在网络字节序中，需要转换为主机字节序。

  4. 返回值：

    • ntohs函数返回一个16位整数值，表示转换后的主机字节序中的数值。
    • 返回的值是以主机字节序表示的原始网络字节序中的整数值。

  5. 使用场景：

    • 在网络编程中，当从网络中接收到一个16位整数时，通常需要使用ntohs函数将其转换为主机字节序，以便在本地进行处理。
    • 同样，在需要将本地数据发送到网络时，也可以使用htons函数将主机字节序转换为网络字节序后再发送。

总之，ntohs函数在网络编程中扮演着重要的角色，用于处理网络数据的字节序转换，确保数据在不同主机之间的正确传输和解释。

recv()

recv函数是用于接收数据的系统调用函数，常用于网络编程中的套接字通信。以下是关于recv函数的解释：

  1. 函数原型：

    • sockfd：指定接收数据的套接字描述符，即需要从哪个套接字接收数据。
    • buf：用于接收数据的缓冲区指针，接收到的数据会被存储到这个缓冲区中。
    • len：指定接收数据的最大长度，即缓冲区的大小。
    • flags：接收标志，可以指定一些接收数据的选项，通常设置为0。

  2. 函数功能：

    • recv函数用于从已连接套接字（如TCP套接字）或无连接套接字（如UDP套接字）接收数据。
    • 当有数据到达时，recv函数会将数据读取到指定的缓冲区buf中，并返回接收到的字节数。
    • 如果没有数据可接收，且对方已关闭连接，则recv函数可能返回0（表示对端已关闭连接）。
    • 如果发生错误，recv函数返回-1，并设置全局变量errno以指示错误的具体原因。

  3. 参数说明：

    • sockfd通常是通过socket函数创建的套接字描述符。
    • buf是一个指向存储接收数据的内存缓冲区的指针。
    • len指定了缓冲区的大小，用于限制接收数据的最大长度。
    • flags通常设置为0，表示普通的接收操作。

  4. 返回值：

    • 成功接收数据时，返回接收到的字节数（大于0）。
    • 对方已关闭连接时，返回0。
    • 出现错误时，返回-1，并设置errno以指示具体的错误类型。

在网络编程中，recv函数通常与send函数配对使用，用于实现数据的收发功能。要注意处理返回值，特别是处理接收到的数据长度以及错误情况，确保程序能够正确地处理数据和连接状态。

strtok()

strtok函数是C语言标准库中的字符串处理函数，用于将字符串分割成一系列子字符串。下面是关于strtok函数的解释：

  1. 函数原型：

    • str：指向要分割的字符串的指针，第一次调用时传入需要分割的字符串，后续调用时传入NULL，用于继续从上次停止的地方继续分割。
    • delim：指定分割字符串的分隔符字符集合，即用于确定分割位置的字符。

  2. 函数功能：

    • strtok函数将字符串str分割成一系列子字符串，每个子字符串由分隔符delim分隔。
    • 第一次调用时，将str指向的字符串进行分割，并返回第一个子字符串的指针。
    • 后续调用时，将str参数设为NULL，strtok将继续从上次停止的位置继续分割，直到遇到新的分隔符或字符串结束。
    • 当没有更多的子字符串可供分割时，strtok返回NULL，表示分割结束。

  3. 注意事项：

    • strtok函数会修改传入的字符串，即将分隔符替换为NULL字符。因此，原始字符串会被修改。
    • 如果需要保留原始字符串，可以将其复制到另一个缓冲区中进行分割操作。
    • 在多线程环境下使用strtok函数可能会出现问题，因为它使用了静态变量来记录分割位置。

sscanf()

sscanf 函数是C语言标准库 <stdio.h> 中的格式化输入函数，用于从字符串中按照指定格式解析数据并赋值给变量。下面是关于 sscanf 函数的解释：

函数原型：

  • str：要解析的输入字符串。
  • format：格式化字符串，指定了 str 中要解析的数据格式。
  • ...：可变参数，用于存储解析出的数据，根据 format 中指定的格式，依次对应输出参数。

函数功能：

  • sscanf 函数从字符串 str 中读取数据，并按照 format 中的格式进行解析。
  • 解析过程会根据 format 中的格式说明符，从输入字符串中提取数据，然后将解析出的数据存储到相应的变量中。

返回值：

  • 返回成功匹配并赋值的参数个数。
    • 如果成功匹配了所有指定的格式说明符并成功赋值，则返回成功赋值的参数个数；
    • 如果发生解析错误或无法匹配所有指定的格式说明符，则返回匹配成功的参数个数，可能小于指定的参数个数。

常用的格式说明符：

  • %d：解析一个十进制整数。
  • %f、%lf：解析一个浮点数，%f 表示 float 类型，%lf 表示 double 类型。
  • %c：解析一个字符。
  • %s：解析一个字符串，遇到空白字符（空格、制表符等）时停止。
  • %x、%X：解析一个十六进制整数。
  • %o：解析一个八进制整数。
  • %u：解析一个无符号十进制整数。
  • %[^delimiter]：解析一段字符序列，直到遇到指定的分隔符为止。

  • sscanf 函数与 scanf 函数类似，但 sscanf 从字符串中读取数据，而 scanf 从标准输入流（键盘）中读取数据。
  • 使用 sscanf 函数时要确保 format 字符串和 str 字符串的格式匹配，否则可能导致解析失败或数据错误。
  • 如果 sscanf 解析失败或无法匹配所有指定的格式说明符，会返回解析成功的参数个数，这个值可能小于指定的参数个数。

总之，sscanf 函数是用于从字符串中解析数据的强大工具，可根据指定的格式将字符串中的数据提取出来并存储到相应的变量中。

sprintf()

sprintf 是C语言标准库 <stdio.h> 中的一个函数，用于将格式化的数据写入字符串中。下面是关于 sprintf 函数的解释：

函数原型：

  • str：要写入的目标字符串的地址。
  • format：格式化字符串，指定了要写入到 str 中的内容和格式。
  • ...：可变参数，用于替换格式字符串中的占位符。

函数功能：

  • sprintf 函数根据格式化字符串 format 中的格式说明符，将后续的参数替换并写入到目标字符串 str 中。
  • 它的功能类似于 printf 函数，但是 printf 函数是将格式化的数据输出到标准输出流（通常是控制台），而 sprintf 函数将格式化的数据输出到字符串中。
  • sprintf 函数会在目标字符串的末尾添加一个 null 结束符，以表示字符串的结束。

格式说明符：

  • 格式说明符以百分号 % 开始，后面跟着一个字符，用于指定要写入的数据的类型和格式。
  • 常见的格式说明符包括 %d（整数）、%f（浮点数）、%s（字符串）等。

strstr()

strstr 函数是C语言标准库 <string.h> 中的函数，用于在一个字符串中搜索指定子串的位置。下面是关于 strstr 函数的解释：

函数原型：

  • haystack：要在其中搜索的字符串，也就是被搜索的源字符串。
  • needle：要搜索的子串，也就是需要在源字符串中寻找的目标子串。

函数功能：

  • strstr 函数在字符串 haystack 中搜索子串 needle，如果找到了，则返回第一次出现该子串的位置的指针；如果未找到，则返回 NULL。
  • 如果 needle 是空字符串（即 needle 的第一个字符为终止符 \0），则 strstr 返回 haystack 的指针。

  • strstr 函数返回的指针指向 haystack 中第一次出现 needle 的位置，如果 needle 在 haystack 中多次出现，则返回的是第一次出现的位置。
  • 如果 needle 是空字符串，strstr 返回 haystack 的指针。

strncmp()

strncmp 函数是C语言标准库 <string.h> 中的函数，用于比较两个字符串的前n个字符是否相等。下面是关于 strncmp 函数的解释：

函数原型：

  • str1：要比较的第一个字符串。
  • str2：要比较的第二个字符串。
  • n：要比较的最大字符数。

函数功能：

  • strncmp 函数比较两个字符串 str1 和 str2 的前 n 个字符。
  • 如果两个字符串在前 n 个字符上完全相等，则返回值为 0。
  • 如果两个字符串在前 n 个字符上不相等，则返回值为它们第一个不相等字符的 ASCII 码差值。
  • 如果 n 大于两个字符串中较短的字符串的长度，则比较长度会被限制为两个字符串的实际长度。

注意事项：

  • strncmp 函数比较的是字符串的前 n 个字符，如果要比较整个字符串是否相等，可以将 n 设置为两个字符串中较长的那个字符串的长度。
  • 如果两个字符串的前 n 个字符都相等，但其中一个字符串长度小于 n，则被认为小的字符串更小。

Socket

socket技术详解（看清socket编程） (csdn.net)

Socket（套接字）是计算机网络编程中的一种通信机制，它允许不同计算机上的程序通过网络进行通信。

Socket允许客户端和服务器之间在网络上进行数据交换。它通过一种类似于文件I/O的方式，提供了一种通用的编程接口，使得开发者可以方便地在网络上发送和接收数据。

在Socket编程中，有两种常用的通信模式：

  • TCP（传输控制协议）和UDP（用户数据报协议）。
  • TCP提供可靠的、面向连接的通信服务，数据传输过程中会进行差错检测和重传；

而UDP则是一种无连接的通信服务，它更加轻量级，适用于一些实时性要求较高、对数据完整性要求不那么严格的场景。

Socket编程在网络应用开发中非常常见，它可以用于构建各种类型的网络应用，包括Web服务器、聊天程序、游戏服务器等。

流程

套接字（Socket）是在网络通信中进行数据交换的一种方式，其流程通常包括以下几个步骤：

  1. 创建套接字（Socket Creation）：

    • 在通信的两端（通常是客户端和服务器端）分别创建套接字。这些套接字可以是基于流（TCP）或数据报（UDP）的，取决于所需的通信方式。

  2. 绑定地址（Bind）（可选）：

    • 服务器端通常需要将套接字与一个特定的地址（IP地址和端口号）绑定在一起，以便客户端能够连接到该地址。客户端通常不需要绑定地址，而是直接连接到服务器。

  3. 监听连接请求（Listen）（仅服务器端）：

    • 服务器端在准备好接受连接之后，调用listen函数开始监听连接请求。此时，服务器进入等待连接的状态。

  4. 接受连接（Accept）（仅服务器端）：

    • 当客户端发送连接请求时，服务器调用accept函数来接受该连接。一旦连接建立成功，服务器会创建一个新的套接字用于与客户端进行通信，并返回该套接字的描述符。

  5. 发起连接（Connect）（仅客户端）：

    • 客户端通过调用connect函数向服务器端发起连接请求。在成功建立连接之后，客户端也会创建一个新的套接字用于与服务器进行通信。

  6. 数据交换（Data Exchange）：

    • 一旦连接建立成功，客户端和服务器端就可以通过各自的套接字进行数据交换。对于基于TCP的通信，数据以流的形式进行传输，保证了数据的顺序和可靠性；而对于基于UDP的通信，数据以数据报的形式进行传输，更加轻量但不保证数据的顺序和可靠性。

  7. 关闭连接（Close Connection）：

    • 当通信结束时，客户端和服务器端分别调用close函数关闭套接字，释放资源并结束通信。

总的来说，套接字的流程包括创建套接字、绑定地址（可选）、监听连接请求（仅服务器端）、接受连接（仅服务器端）、发起连接（仅客户端）、数据交换以及关闭连接。这些步骤构成了网络通信中的基本流程。

OSI和TCP/IP对比

Socker层位置

在tcp/ip协议中，tcp通过三次握手建立起一个tcp的链接，大致如下：

  • 第一次握手：客户端尝试连接服务器，向服务器发送syn包，syn=j，客户端进入SYN_SEND状态等待服务器确认。
  • 第二次握手：服务器接收客户端syn包并确认（ack=j+1），同时向客户端发送一个SYN包（syn=k），即SYN+ACK包，此时服务器进入SYN_RECV状态。
  • 第三次握手：客户端收到服务器的SYN+ACK包，向服务器发送确认包ACK(ack=k+1），此包发送完毕，客户端和服务器进入ESTABLISHED状态，完成三次握手。

知识点

服务器端：

其过程是首先服务器方要先启动，并根据请求提供相应服务：

（1）打开一通信通道并告知本地主机，它愿意在某一公认地址上的某端口（如FTP的端口可能为21）接收客户请求；

（2）等待客户请求到达该端口；

（3）接收到客户端的服务请求时，处理该请求并发送应答信号。接收到并发服务请求，要激活一新进程来处理这个客户请求（如UNIX系统中用fork、exec）。新进程处理此客户请求，并不需要对其它请求作出应答。服务完成后，关闭此新进程与客户的通信链路，并终止。

（4）返回第（2）步，等待另一客户请求。

（5）关闭服务器

客户端：

（1）打开一通信通道，并连接到服务器所在主机的特定端口；

（2）向服务器发服务请求报文，等待并接收应答；继续提出请求……

（3）请求结束后关闭通信通道并终止。

从上面所描述过程可知：

（1）客户与服务器进程的作用是非对称的，因此代码不同。

（2）服务器进程一般是先启动的。只要系统运行，该服务进程一直存在，直到正常或强迫终止。

参数

af指定通信发生的区域：AF_UNIX、AF_INET、AF_NS等，而DOS、WINDOWS中仅支持AF_INET，它是网际网区域。因此，地址族与协议族相同。

（3）三是原始式套接字(SOCK_RAW)该接口允许对较低层协议，如IP、ICMP直接访问。常用于检验新的协议实现或访问现有服务中配置的新设备。

protocol说明该套接字使用的特定协议，如果调用者不希望特别指定使用的协议，则置为0，使用默认的连接模式。

根据这三个参数建立一个套接字，并将相应的资源分配给它，同时返回一个整型套接字号。因此，socket()系统调用实际上指定了相关五元组中的“协议”这一元。

设置套接字选项——setsockopt()

参数

  • sockfd：指定需要设置选项的套接字文件描述符。
  • level：指定选项所在的协议层或者协议族。常见的值包括：
    • SOL_SOCKET：通用套接字选项，适用于所有套接字类型。
    • IPPROTO_TCP：TCP协议特定选项。
    • IPPROTO_IP：IP协议特定选项。
    • 其他特定协议的选项层级。
  • optname：指定需要设置的选项名称，是一个整数常量。常见的选项名称包括：
    • SO_REUSEADDR：允许重用地址。
    • SO_KEEPALIVE：启用或禁用保活功能。
    • SO_LINGER：控制close()行为。
    • SO_RCVBUF、SO_SNDBUF：设置接收缓冲区和发送缓冲区大小。
    • 其他协议特定的选项。
  • optval：指向包含选项值的缓冲区的指针，类型为void*。选项的值可以是任何类型，通常是一个特定的结构体，根据选项的不同而不同。
  • optlen：指定选项值的长度。

setsockopt()函数的作用是将指定套接字的选项设置为特定的值，以便控制套接字的行为。根据选项的不同，可以控制套接字的重用、保活机制、缓冲区大小等。

一般来说，setsockopt()函数的返回值为0表示设置成功，-1表示设置失败，具体的错误信息可以通过errno全局变量获取。

SOCKADDR_IN——定义地址结构体

SOCKADDR_IN 是 Winsock 库中用于表示 IPv4 地址和端口号的结构体，通常用于套接字编程。以下是 SOCKADDR_IN 结构体的成员和用法解释：

  1. sin_family：

    • 表示地址族（Address Family），通常设置为 AF_INET，表示 IPv4 地址族。

  2. sin_port：

    • 表示端口号，使用网络字节序（大端字节序）表示，即端口号在存储时采用的字节顺序。

  3. sin_addr：

    • 表示 IPv4 地址的结构体，通常包含在 in_addr 结构体中。
    • sin_addr 中的具体成员根据不同的操作系统和库的实现而有所不同：
      • 在 Windows 平台上，通常使用 S_un 联合体来访问 IPv4 地址，具体有 S_addr 和 S_un_b 等成员，用于存储 IP 地址的不同表示形式。

  4. sin_zero：

    • 用于填充结构体，使其与 SOCKADDR 结构的大小相同。

建立套接字连接──connect()与accept()

这两个系统调用用于完成一个完整相关的建立，其中connect()用于建立连接。accept()用于使服务器等待来自某客户进程的实际连接。

connect()

参数
s是欲建立连接的本地套接字描述符。

name指出说明对方套接字地址结构的指针。

namelen指出对方套接字地址长度。

返回

如果没有错误发生，connect()返回0。否则返回值SOCKET_ERROR。在面向连接的协议中，该调用导致本地系统和外部系统之间连接实际建立。

由于地址族总被包含在套接字地址结构的前两个字节中，并通过socket()调用与某个协议族相关。因此bind()和connect()无须协议作为参数。

accept()

参数

s为本地套接字描述符，在用做accept()调用的参数前应该先调用过listen()。

addr 指向客户方套接字地址结构的指针，用来接收连接实体的地址。addr的确切格式由套接字创建时建立的地址族决定。

addrlen 为客户方套接字地址的长度（字节数）。

返回

如果没有错误发生，accept()返回一个SOCKET类型的值，表示接收到的套接字的描述符。否则返回值INVALID_SOCKET。

accept()用于面向连接服务器。参数addr和addrlen存放客户方的地址信息。

调用前，参数addr 指向一个初始值为空的地址结构，而addrlen 的初始值为0；

调用后，服务器等待从编号为s的套接字上接受客户连接请求，而连接请求是由客户方的connect()调用发出的。

当有连接请求到达时，accept()调用将请求连接队列上的第一个客户方套接字地址及长度放入addr 和addrlen，并创建一个与s有相同特性的新套接字号。

套接字可用于处理服务器并发请求。

四个套接字系统调用，socket()、bind()、connect()、accept()，可以完成一个完全五元相关的建立。socket()指定五元组中的协议元，它的用法与是否为客户或服务器、是否面向连接无关。bind()指定五元组中的本地二元，即本地主机地址和端口号，其用法与是否面向连接有关：在服务器方，无论是否面向连接，均要调用bind()，若采用面向连接，则可以不调用bind()，而通过connect()自动完成。若采用无连接，客户方必须使用bind()以获得一个唯一的地址。
