标签中id和name的作用和区别

  1. 个人最喜欢的答案
    ID 是 以 #定义的CSS样式
    也可以用JS获取来控制 getElementById(这里是ID) 来获取
    name 很多了，就是给当前标签或元素指定名称，也可以用JS来控制值，form提交后获取的时候就需要获取name名称。
  2. 次之的答案
    id 一般用于css和js中引用,name用于表单提交,只有加了name属性的标签元素才会提交到服务器
  3. 不喜欢但可以参考的答案
    id要符合标识的要求,比如大小写敏感,最好不要包含下划线(因为不兼容CSS)。而name基本上没有什么要求,甚至可以用数字。ID就像是一个人的身份证号码，而Name就像是他的名字，ID显然是唯一的，而Name是可以重复的。
    标签规定用户可输入数据的输入字段。
    根据不同的 type 属性，输入字段有多种形态。输入字段可以是文本字段、复选框、密码字段、单选按钮、按钮等等。
    id：作为标签的唯一标识。
    name：作为可与服务器交互数据的HTML元素的服务器端的标示。

分类

块元素

  1. 独占一行
  2. 高宽，内外边距都可以改变
  3. 宽度默认是容器的100%
  4. 是一个容器及盒子
    • 注意：
      • 文字类的元素不能使用块级元素
      • < p >主要用于存放文字，不能放块级元素，特别是< div >
      • < h1 >~< h6 >都是文字类块级标签，里面也不能放其他块级元素

行内元素

  1. 一行可以多个
  2. 高、宽直接设置时无效的
  3. 默认宽度就是它本身内容的宽度
  4. 只能容纳文本和其他行内元素
    • 注意：
      • 链接里面不能放链接
      • < a >可以放块级元素，但是给< a > 转换一下块级模式最安全

行内块元素

  1. 一行可以多个，中间会有空隙
  2. 可以设置宽、高、内边距
    1. < img / >
    2. < input />
    3. < td >

元素显示模式切换

  • 转换为块元素：display:block;
  • 转换为行内元素：display:inline;
  • 转换为行内块元素：display:inline-block;

定位模式——子绝父相

  • 相对定位——relative：

    • 按照自己原来的位置来移动
    • 给绝对定位当爹
    • 保留原来的位置——不脱标

  • 绝对定位——absolute：

    • 按照祖先元素来移动位置
    • 没有父元素，或者父元素没有定位模式，按照浏览器文档来移动位置
    • 祖先元素有定位，按照最近一级有定位来移动位置
    • 保留原来位置——脱标

  • 固定定位——fixed：

    • 与父元素没有关系

    • 不随着滚动条滚动

    • 按照浏览器可视窗口移动位置

    • 不保留原来位置——脱标

    URL

    统一资源定位符，简称网址

    • 协议：超文本传输协议，规定浏览器和服务器之间传输数据的格式

    URL查询参数

    • 语法：http://xxx.com/xx?参数名1=值1&参数名2=值2

    HTTP协议——请求报文

    浏览器按照HTTP协议要求的格式，发送给服务器的内容

    1. 请求行：请求方法，URL，协议
    2. 请求头：以键值对的格式携带的附加信息，比如：Content-Type
    3. 空行：分隔请求头，空行之后是发送给服务器的资源
    4. 请求体：发送的资源

    在浏览器的网络—>Fetch/XHR，里可以看到请求报文（内容在载荷里）

    HTTP协议——响应报文

    服务器按照HTTP协议要求的给是，返回给浏览器的内容

    1. 响应行（状态行）：协议、HTTP响应状态码、状态信息
    2. 响应头：以键值对的格式携带的附加信息，比如：Content-Type
    3. 空行：分隔响应头，空行之后是服务器返回的资源
    4. 响应体：返回的资源
    • 1xx：信息
    • 2xx：成功
    • 3xx：重定向消息
    • 4xx：客户端错误
    • 5xx：服务端错误

    在浏览器的网络—>Fetch/XHR—>响应，里可以看到响应报文
